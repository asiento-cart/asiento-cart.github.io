1. Введение и определения

1.1. Терминология:

  • "Гильдия" - добровольное объединение физических лиц Asiento Cart
  • "Участник" - любое лицо, использующее сервисы платформы
  • "ПДн" - персональные данные согласно 152-ФЗ
  • "Цифровые активы" - криптовалюты, рассматриваемые как имущество (ст.128 ГК РФ)

1.2. Правовая основа:

  • Конституция РФ (ст.23, 24) - право на неприкосновенность частной жизни
  • 152-ФЗ "О персональных данных"
  • 259-ФЗ "О цифровых финансовых активах"
  • Налоговый кодекс РФ (ст.208, 224)

2. Техническая инфраструктура

2.1. Хостинг и хранение данных:

Сайт-визитка размещен на GitHub Pages (политика конфиденциальности GitHub)

  • Не использует cookies
  • Не содержит форм сбора данных
  • Статистика посещений доступна только в агрегированном виде

2.2. Телеграм-бот:

  • Хранит только:
    • Хеши транзакций
    • Анонимные ID пользователей
    • Временные метки операций

3.1. Принципы работы с ПДн:

3.2. Гарантии конфиденциальности:

Гильдия НЕ является оператором ПДн согласно 152-ФЗ (ст.3)

  • Все сделки между участниками - их личная ответственность
  • Рекомендуется:
    • Использовать одноразовые email
    • Создавать отдельные кошельки для операций
    • Не передавать паспортные данные

4. Финансовые аспекты и налоги

4.1. Налоговая ответственность:

Участники обязаны:

  • ✓ Вести учет всех операций
  • ✓ Самостоятельно исчислять НДФЛ (13-15%)
  • ✓ Подавать декларацию 3-НДФЛ до 30 апреля

Гильдия предоставляет:

  • * Шаблоны для учета операций
  • * Актуальные налоговые ставки
  • * Консультации по заполнению форм

4.2. Ограничения:

❗ Запрещено:

  • Использовать платформу для обхода санкций
  • Проводить операции на сумму свыше 600 тыс. руб. без верификации
  • Сокрытие доходов от налоговых органов

5. Ответственность и взаимодействие участников

Все сделки на платформе осуществляются исключительно между физическими лицами по схеме P2P (peer-to-peer)

5.1. Принцип персональной ответственности

Гильдия выступает только техническим посредником, предоставляя:

  • Инструменты для коммуникации
  • Систему репутации
  • Шаблоны договоренностей

Юридический статус операций:

Каждая сделка считается частным соглашением между двумя физическими лицами в соответствии со статьей 421 ГК РФ "Свобода договора"

5.2. Ограничения ответственности Гильдии

Гильдия не несет ответственности за:

  • Выполнение обязательств по сделкам
  • Подлинность предоставляемых участниками данных
  • Налоговые последствия операций
  • Технические сбои при передаче активов

5.3. Механизм взаимодействия участников

Перед сделкой:

  • Рекомендуется заключить простую письменную форму договора
  • Проверить репутацию контрагента через систему рейтингов
  • Уточнить условия перевода активов

Во время сделки:

  • Использовать гарант-сервисы (если предусмотрено)
  • Фиксировать все этапы переписки
  • Сохранять хеши транзакций

После сделки:

  • Обменяться электронными актами приема-передачи
  • Оставить отзыв о контрагенте
  • Сохранить доказательства операции на 3 года

6.1. Изменения в политике:
